Union Find

コードについての説明

Union Find は推移的な関係をうまく保持するデータ構造である.
競技プログラミングでは基本的なアルゴリズムなので, 説明は蟻本またはこのスライドにゆずる.
すでに知っている方は計算量が O(α(n)) (α(n)はアッカーマンの逆関数) になる証明を追ってみるとおもしろいかもしれません. 証明は Tarjan の元論文よりも, 分かりやすく書かれている方の記事を見たほうが良いです.

時間計算量: クエリ O(α(n)) (α(n)はアッカーマンの逆関数)

コード

class UnionFind {
private:
    int sz;
    vector<int> par, size_;
public:
    UnionFind(){}
    UnionFind(int node_size) : sz(node_size), par(sz), size_(sz, 1){
        iota(par.begin(), par.end(), 0);
    }
    int find(int x){
        if(par[x] == x) return x;
        else return par[x] = find(par[x]);
    }
    void unite(int x,int y){
        x = find(x), y = find(y);
        if(x == y) return;
        if(size_[x] < size_[y]) swap(x,y);
        par[y] = x;
        size_[x] += size_[y];
    }
    int size(int x){
        x = find(x);
        return size_[x];
    }
    bool same(int x,int y){
        return find(x) == find(y);
    }
};
