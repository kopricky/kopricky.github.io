Minimum Cost Flow Algorithm with Negative Cycles

個人的メモ

負の有向閉路を含まないグラフの場合はコストを距離と見て最短距離に沿って増加フローを流し続け, 増加フローが無くなったら終了するというようなアルゴリズムであったが, このアルゴリズムではまず始めに実行可能解となるフロー $f$ を最大流問題を解くアルゴリズムを用いて求める.
そのあと残余ネットワーク内の閉路でその平均の長さが最小であるものに沿って増加フローを流し続け, 閉路の平均長が非負となったら終了するというようなアルゴリズムである. ここで最小平均長の閉路は こちらのアルゴリズム を用いて $\O (mn)$ 時間で求めることができる.

(最適性について)
実行可能解となるフロー $f$ を流したときに, 残余ネットワーク上で負の有向閉路が存在しないことと $f$ が最小費用流となっていることとの同値性が自明でないと思う(特に左から右)のでそれについての備忘録.
まず (簡単版の)最小費用流のアルゴリズム でも記したように最小費用流を線形計画問題と見てその双対問題および相補性条件を書き下すと, 相補性条件は残余ネットワーク $G_f$ 上でポテンシャルが存在すること(ある $p$ : $V \rightarrow \mathbb{R}$ が存在し, $f(e) < e.cap$ であるすべての辺 $e = (u, v)$ について $e.cost + p(u) + p(v) \ge 0$ が成り立つ) であったことを思い出したい. グラフに負の有向閉路が存在しないならある頂点 $s$ から各頂点への最短距離をそのままポテンシャルとすればポテンシャルの構成はできる(負の有向閉路が存在しない → ポテンシャルが存在). 最小平均長閉路に沿ってフローを流して更新している間フロー $f$ は常に主問題の実行可能解であり, また負の有向閉路が存在しない残余ネットワーク上においては上記のように双対問題の実行可能解 $p$ でポテンシャルの条件を満たすものを取ることができる. つまり主問題、双対問題それぞれの実行可能解 $f, p$ について相補性条件が成り立つ. よって確かに残余ネットワーク上に負の有向閉路が存在しない場合フロー $f$ は最適解であることがわかる.

(計算量について)
最小平均長閉路長のみからごりごり証明を行っているもの(Korte, Vygen 本) もあるがポテンシャルの $\epsilon$-最適性の観点から示す方が断然良いと思う.
あまりネット上には良い感じの記事が上がっていないがこの Lecture Note(11 ~ 14) は簡潔にまとまっていると思う.
