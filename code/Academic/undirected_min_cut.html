Undirected Mincut

コードについての説明

無向グラフの大域最小容量カットを求めるアルゴリズム. 愚直に n-1 回 s,t 最大フローを求めるよりも高速に解くことができ計算量は O(mn + n^2log(n)) である.
ある頂点 v_1 から始めて今の頂点集合 S と一番連結度の高い頂点を S に加えていくというように貪欲に取っていく順序を MA(最大隣接) 順序と呼ぶ.
このように頂点を並び替えることで効率よく大域最小容量カットを求めることができる. これは(対称)劣モジュラ関数の最小化というより一般的な視点で見るととてもおもしろい(Queyranne 1998).
元論文は Nagamochi and Ibaraki 1992
大域最小容量カットは他にも, 何も考えずグラフの辺をランダムに縮約していき残り 2 点になったときの連結度を最小容量カットとするというような一連の操作を O(n^2) くらい繰り返せば高確率で正しい値を得られるというようなモンテカルロアルゴリズムも存在する(Randomized Algorithm などにのっている).
最後に上記の計算量は MA 順序の計算量にフィボナッチヒープを用いたものであり, 以下の実装で少し遅い O(n^3) である.

時間計算量: O(n^3)

コード

template<typename T> class UndirectedMinCut
{
private:
    int V;
    vector<vector<T> > G;
    vector<int> order;
public:
    UndirectedMinCut(int node_size) : V(node_size), G(V, vector<T>(V,0)), order(V){
        iota(order.begin(),order.end(),0);
    }
    void add_edge(int u,int v,T cap){
        G[u][v] += cap, G[v][u] += cap;
    }
    T solve(){
        T ans = numeric_limits<T>::max();
        for(int i = V; i > 1; i--){
            vector<T> val(i,0);
            vector<int> new_order(i-1);
            for(int j = 0; j < i-1; j++){
                int mx_id = max_element(val.begin(), val.end()) - val.begin();
                new_order[j] = order[mx_id];
                val[mx_id] = -1;
                for(int k = 0; k < i; k++){
                    if(val[k] >= 0){
                        val[k] += G[order[mx_id]][order[k]];
                    }
                }
            }
            int last = max_element(val.begin(),val.end()) - val.begin();
            int last_ver = order[last];
            int nx_last = new_order.back();
            for(int j = 0; j < (int)new_order.size()-1; j++){
                G[nx_last][new_order[j]] += G[last_ver][new_order[j]];
                G[new_order[j]][nx_last] += G[new_order[j]][last_ver];
            }
            order = new_order;
            ans = min(ans,val[last]);
        }
        return ans;
    }
};
