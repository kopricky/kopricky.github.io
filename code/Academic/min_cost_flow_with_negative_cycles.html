Minimum Cost Flow Algorithm with Negative Cycles

コードについての説明

負の有向閉路を含むグラフに対する最小費用流問題を解くアルゴリズム. 正確には任意のコスト関数, 容量関数に対して正しい $s$-$t$ 間最小費用流を返すアルゴリズムである. 複数の source および sink がある場合も超頂点 $S$, $T$ を用意して $S$ から各 source, 各 sink から $T$ へ辺を張ることで変換可能.
負の有向閉路を含まないグラフの場合はコストを距離と見て最短距離に沿って増加フローを流し続け, 増加フローが無くなったら終了するというようなアルゴリズムであったが, このアルゴリズムではまず始めに実行可能解となるフロー $f$ を最大流問題を解くアルゴリズムを用いて求める.
そのあと残余ネットワーク内の閉路でその平均の長さが最小であるものに沿って増加フローを流し続け, 閉路の平均長が非負となったら終了するというようなアルゴリズムである. ここで最小平均長の閉路は こちらのアルゴリズム を用いて $\O (mn)$ 時間で求めることができる.
このアルゴリズムは弱多項式時間であるだけでなく, さらには強多項式時間であることも後の解析で明らかになり, 計算量は $\O (\min (m^2 n^2 \log (n |c_{\min}|), m^3 n^2 \log n))$ となる($c_{\min}$ は辺コストの最小値を表し, また前者は辺のコストが整数の場合にのみ成り立つ).
実測は速くなく min25 さんの凄そうなライブラリ など他の方の実装のほうが速いと思う.

(最適性について)
実行可能解となるフロー $f$ を流したときに, 残余ネットワーク上で負の有向閉路が存在しないことと $f$ が最小費用流となっていることとの同値性が自明でないと思う(特に左から右)のでそれについての備忘録.
まず (簡単版の)最小費用流のアルゴリズム でも記したように最小費用流を線形計画問題と見てその双対問題および相補性条件を書き下すと, 相補性条件は残余ネットワーク上でポテンシャルが存在すること(ある $p$ : $V \rightarrow \mathbb{R}$ が存在し, $f(e) < e.cap$ であるすべての辺 $e = (u, v)$ について $e.cost + p(u) + p(v) \ge 0$ が成り立つ) であったことを思い出したい. グラフに負の有向閉路が存在しないならある頂点 $s$ から各頂点への最短距離を Bellman ford を用いて求めることができ, それをそのままポテンシャルとすればポテンシャルの構成はできる(負の有向閉路が存在しない → ポテンシャルが存在). 最小平均長閉路に沿ってフローを流して更新している間フロー $f$ は常に主問題の実行可能解であり, また負の有向閉路が存在しない残余ネットワーク上においては上記のように双対問題の実行可能解 $p$ でポテンシャルの条件を満たすものを取ることができる. つまり主問題、双対問題それぞれの実行可能解 $f, p$ について相補性条件が成り立つ. よって確かに残余ネットワーク上に負の有向閉路が存在しない場合フロー $f$ は最適解であることがわかる.

(計算量について(見づらくなってしまったのでもっといい証明を考えるかなんかする))
辺の容量がすべて整数であることを仮定して計算量が $\O (m^2 n^2 \log (n |c_{\min}|))$ となることを考える.
以下で示すことは $mn$ 回の更新ごとに最小平均長閉路の長さの絶対値は少なくとも $1/2$ 以下になるということである. これが示せたとする. $i$ 回目の更新の際の最小平均長閉路の値を $L_i (< 0)$ とすると, $|L_1|$ は明らかに $|c_{\min}|$ 以下である. そして $k = \Theta(mn \log(n |c_{\min}|))$ 回更新がもし続いたとすると, $|L_k|$ が $1 / n$ より小さくなっているので, 辺の容量が全て整数である事実から $|L_k| = 0$ が言え, アルゴリズムが終了する. ゆえに $\O (mn \log(n |c_{\min}|))$ 回の更新で最小平均長閉路の長さは非負となる.
まず初めに $|L_i| \ge |L_{i+1}|$ は成り立つことが言える. $i$ 回目の増加フローになる閉路を $C_i$ とする. このとき $C_i$ と $C_{i+1}$ の和集合から互いに逆向きとなる辺を除去したグラフ $H$ を考えると, それは disjoint な有向閉路に分解することができる. ここで "$C_i$ は $i$ 回目の時点での最小平均長閉路であること", "$H$ に含まれる辺はすべて $i$ 回目の残余ネットワーク上に存在すること" からすべての有向閉路についてその平均長が $L_i$ 以上となる. よって (グラフ $H$ の辺コストの総和) $\left( = L_i |E(C_i)| + L_{i+1}|E(C_{i+1})| \right)$ $\ge L_k |E(H)|$ が成り立つ. すると $|E(H)| \le |E(C_i)| + |E(C_{i+1})|$ の事実と合わせて $L_i \le L_{i+1}$ が導かれて, $|L_i| \ge |L_{i+1}|$ となる.
次に, 各更新で少なくとも 1 つの辺が saturated になり残余ネットワークから消えることから, $i$ 回目から $i + m$ 回目に更新された $m + 1$ 個の閉路について考えるとその中のある組 $j, k(j < k)$ で $C_j, C_k$ は互いに逆向きとなる辺を有している($\exists e \in C_j$, $e.rev \in C_k$) という事実が成り立つことが分かる. このような $j, k$ について $|L_j| \ge (n / (n - 2)) \times |L_k|$ が成り立つことを示す(つまり連続する $m + 1$ 回の更新の間に最小平均長閉路の長さの絶対値が $(n - 2) / n$ の factor で減少していることを示す).
先程同様に $C_j$ と $C_k$ の和集合から互いに逆向きとなる辺を除去したグラフ $H$ について辺コストの総和を考える. このとき $j$ として "$C_j, C_k$ は互いに逆向きとなる辺を有している" という条件を満たすもののうち最大のものを取ってきておくと議論が早い. なぜならこのとき $C_k$ 内にあって $C_j$ 内の辺と互いに逆向きとならないような辺はすべて $j$ 回目の更新時点で残余ネットワーク上に存在することが言えるからである. 言い換えるとグラフ $H$ に含まれる辺はすべて $j$ 回目の残余ネットワーク上に存在することが言えるので, 先ほどと同様の議論より (グラフ $H$ の辺コストの総和) $\left( = L_j |E(C_j)| + L_{k}|E(C_{k})| \right)$ $\ge L_j |E(H)|$ が成り立つ. ただ今回は $C_j$ 内の辺と $C_k$ 内の辺の内少なくとも $1$ つは打ち消し合うという事実が分かっているので $|E(H)| \le |E(C_j)| + |E(C_k)| - 2 \le |E(C_j)| + ((n - 2) / n) \times |E(C_k)|$ のように辺数を先程より厳密に評価することができる. 以上をあわせて, $L_j \times (|E(C_j)| + ((n - 2) / n) \times |E(C_k)|) \le L_j \times |E(H)| \le$ (グラフ $H$ の辺コストの総和) $= L_j \times |E(C_j)| + L_k \times |E(C_k)|$ から $((n - 2) / n) \times L_j \le L_k$, つまり $|L_j| \ge (n / (n-2)) \times |L_k|$ が分かる.
ゆえに示した $2$ つの不等式から $|L_i| \ge |L_j| \ge (n / (n - 2)) \times |L_k| / (n - 2) \ge (n / (n - 2)) \times |L_{i+m}|$ が成り立つので、 連続する $m + 1$ 回の更新の間に少なくとも $|L|$ は $(n - 2) / n$ factor 減少する. この反復を $n$ 回繰り返すと $((n - 2) / n)^n < e^{1 / 2} < 1 / 2$ という事実から, $mn$ 回更新した後には $|L|$ は少なくとも $1 / 2$ 倍以下になっていることが言える.

時間計算量: $\O (\min (m^2 n^2 \log (n |c_{\min}|), m^3 n^2 \log n))$ (ただし $c_{\min}$ は辺コストの最小値を表し, また前者は辺のコストが整数の場合にのみ成り立つ)

コード

template<class edge> class Dinic {
private:
    using Cat = decltype(edge::cap);
    using Cot = decltype(edge::cost);
    vector<vector<edge> >& G;
    const int V;
    vector<int> level, iter;
    void bfs(const int s){
        fill(level.begin(), level.end(), -1);
        queue<int> que;
        level[s] = 0;
        que.push(s);
        while(!que.empty()){
            const int v = que.front();
            que.pop();
            for(const auto& e : G[v]){
                if(e.cap > 0 && level[e.to] < 0){
                    level[e.to] = level[v] + 1;
                    que.push(e.to);
                }
            }
        }
    }
    Cat dfs(const int v, const int t, const Cat f, Cot& res){
        if(v == t) return f;
        for(int& i = iter[v]; i < (int)G[v].size(); ++i){
            edge& e = G[v][i];
            if(e.cap > 0 && level[v] < level[e.to]){
                Cat d = dfs(e.to, t, min(f, e.cap), res);
                if(d > 0){
                    e.cap -= d;
                    G[e.to][e.rev].cap += d;
                    res += d * e.cost;
                    return d;
                }
            }
        }
        return 0;
    }

public:
    Dinic(vector<vector<edge> >& _G) : G(_G), V((int)G.size()), level(V), iter(V){}
    Cot solve(const int s, const int t, Cat f){
        Cot res = 0;
        for(;;){
            bfs(s);
            if(level[t] < 0) return numeric_limits<Cot>::min();
            fill(iter.begin(), iter.end(), 0);
            while(true){
                Cat df = dfs(s, t, f, res);
                if(df > 0){
                    f -= df;
                    if(f == 0) return res;
                }else{
                    break;
                }
            }
        }
    }
};

template<typename CapType, typename CostType> class MinCostFlow
{
public:
    struct edge;
private:
    using Cat = CapType;
    using Cot = CostType;
    const int V;
    const Cot inf;
    vector<vector<Cot> > dp;
    vector<vector<edge*> > prev;
    int minimum_mean_cycle(){
        for(int k = 0; k < V; ++k){
            for(int i = 0; i < V; ++i){
                dp[k+1][i] = inf;
            }
            for(int i = 0; i < V; ++i){
                if(dp[k][i] == inf) continue;
                for(edge& e : G[i]){
                    if(e.cap > 0 && dp[k+1][e.to] > dp[k][i] + e.cost){
                        dp[k+1][e.to] = dp[k][i] + e.cost;
                        prev[k+1][e.to] = &e;
                    }
                }
            }
        }
        if(*min_element(dp[V].begin(), dp[V].end()) == inf) return -1;
        int start = -1;
        Cot tnum = inf / V;
        int tden = 1;
        for(int i = 0; i < V; ++i){
            if(dp[V][i] == inf) continue;
            Cot num = -inf / V;
            int den = 1;
            for(int k = 0; k < V; ++k){
                if(dp[k][i] == inf) continue;
                if(num * (V-k) <= den * (dp[V][i] - dp[k][i])){
                    num = dp[V][i] - dp[k][i], den = V - k;
                }
            }
            if(num * tden < den * tnum){
                tnum = num, tden = den, start = i;
            }
        }
        return (tnum < 0) ? start : -1;
    }
    bool update(Cot& res){
        int start = minimum_mean_cycle();
        if(start < 0) return false;
        int level = V;
        Cat df = numeric_limits<Cat>::max();
        vector<Cat> route;
        vector<int> used(V, -1);
        while(true){
            if(used[start] >= 0){
                for(int i = used[start]; i < (int)route.size(); ++i){
                    df = min(df, route[i]);
                }
                level = V - used[start];
                break;
            }
            used[start] = (int)route.size();
            edge *e = prev[level--][start];
            route.push_back(e->cap);
            start = G[e->to][e->rev].to;
        }
        int cur = start;
        do {
            edge *e = prev[level--][cur];
            edge *reve = &G[e->to][e->rev];
            e->cap -= df, reve->cap += df, res += df * e->cost;
            cur = reve->to;
        }while(level >= 0 && cur != start);
        return true;
    }
public:
    struct edge {
        int to; Cat cap; Cot cost; int rev;
    };
    vector<vector<edge> > G;
    MinCostFlow(const int node_size)
     : V(node_size), inf(numeric_limits<Cot>::max()),
        dp(V+1, vector<Cat>(V, 0)), prev(V+1, vector<edge*>(V)), G(V){}
    void add_edge(const int from, const int to, const Cat cap, const Cot cost){
        G[from].push_back((edge){to, cap, cost, (int)G[to].size()});
        G[to].push_back((edge){from, 0, -cost, (int)G[from].size() - 1});
    }
    Cot solve(const int s, const int t, Cat f){
        Dinic<edge> dn(G);
        Cot res = dn.solve(s, t, f);
        if(res == numeric_limits<Cot>::min()) return res;
        while(update(res));
        return res;
    }
};
