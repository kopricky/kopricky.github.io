Minimum Cost Flow Algorithm with Negative Cycles

コードについての説明

負の有向閉路を含むグラフに対する最小費用流問題を解くアルゴリズム. 正確には任意のコスト関数, 容量関数に対して正しい s-t 間最小費用流を返すアルゴリズムである. 複数の source および sink がある場合も超頂点 S, T を用意して S から各 source, 各 sink から T へ辺を張ることで変換可能.
負の有向閉路を含まないグラフの場合はコストを距離と見て最短距離に沿って増加フローを流し続け, 増加フローが無くなったら終了するというようなアルゴリズムであったが, このアルゴリズムではまず始めに実行可能解となるフロー f を最大流問題を解くアルゴリズムを用いて求める.
そのあと残余ネットワーク内の閉路でその平均の長さが最小であるものに沿って増加フローを流し続け, 閉路の平均長が非負となったら終了するというようなアルゴリズムである. ここで最小平均長の閉路は こちらのアルゴリズム を用いて O(mn) 時間で求めることができる.
このアルゴリズムは弱多項式時間であるだけでなく, さらには強多項式時間であることも後の解析で明らかになり, 計算量は O(min(m^2 n^2 log (n |c_min|), m^3 n^2 log n)) となる(c_min は辺コストの最小値を表し, また前者は辺のコストが整数の場合にのみ成り立つ).
実測は速くなく min25 さんの凄そうなライブラリ など他の方の実装のほうが速いと思う.

(最適性について)
実行可能解となるフロー f を流したときに, 残余ネットワーク上で負の有向閉路が存在しないことと f が最小費用流となっていることとの同値性が自明でないと思う(特に左から右)のでそれについての備忘録.
まず (簡単版の)最小費用流のアルゴリズム でも記したように最小費用流を線形計画問題と見てその双対問題および相補性条件を書き下すと, 相補性条件は残余ネットワーク上でポテンシャルが存在すること(ある p: V → R が存在し, f(e) < e.cap であるすべての辺 e = (u, v) について e.cost + p(u) + p(v) >= 0 が成り立つ) であったことを思い出したい. グラフに負の有向閉路が存在しないならある頂点 s から各頂点への最短距離を Bellman ford を用いて求めることができ, それをそのままポテンシャルとすればポテンシャルの構成はできる(負の有向閉路が存在しない → ポテンシャルが存在). 最小平均長閉路に沿ってフローを流して更新している間フロー f は常に主問題の実行可能解であり, また負の有向閉路が存在しない残余ネットワーク上においては上記のように双対問題の実行可能解 p でポテンシャルの条件を満たすものを取ることができる. つまり f, p について相補性条件が成り立つ. よって確かに残余ネットワーク上に負の有向閉路が存在しない場合フロー f は最適解であることがわかる.
グラフからでも一応考えることはできて, 今残余ネットワーク上で負の有向閉路が存在しないが, フロー f が最適ではなく, それよりコストの小さいフロー f* が存在したとする. このとき f* と f の差分フロー f*(e) - f(e) を考えると, これは source および sink を持たないフローであるので少し端折ると有向閉路をなすフローの集合に分解することができる. すると (f* のコスト) < (f のコスト) である事実から少なくとも 1 つ負の有向閉路が存在することが言え, 矛盾.

(計算量について(見づらくなってしまったのでもっといい証明を考えるかなんかする))
辺の容量がすべて整数であることを仮定して計算量が O(m^2 n^2 log (n |c_min|)) となることを考える.
以下で示すことは mn 回の更新ごとに最小平均長閉路の長さの絶対値は少なくとも 1/2 以下になるということである. これが示せたとする. i 回目の更新の際の最小平均長閉路の値を L_i とすると, |L_1| は明らかに |c_min| 以下である. そして k = O(mn log(n |c_min|)) 回更新がもし続いたとすると, |L_k| が 1 / n より小さくなっているので, 辺の容量が全て整数である事実から |L_k| = 0 が言え, アルゴリズムが終了する.
|L_i| ≧ |L_{i+1}| はまず成り立つ. i 回目の増加フローになる閉路を C_i とする. |L_i| < |L_{i+1}|, つまり L_i ≧ L_{i+1} を仮定すると C_i が i 回目の時点での最小平均長閉路であるという事実から少なくとも C_{i+1} は i 回目の更新で残余ネットワークにできた新たな逆辺を用いているはずである. このとき C_i と C_{i+1} の和集合から打ち消し合う辺を除去したグラフを考えると, それは有向オイラーグラフになっている. よって有向閉路に分解することができる. すべての有向閉路の平均長が L_i 以上とするとそのグラフの総和を考えたとき L_i > L_{i+1} に矛盾するので, 有向閉路の中には必ず平均長が L_i より小さいものが含まれていて, またその有向閉路は i 回目の更新時点で既に存在することも分かる. ゆえに C_i が i 回目の更新時点で平均長が最小の閉路であるという事実に矛盾する.
次に, 各更新で少なくとも 1 つの辺が saturated になり残余ネットワークから消えることから, i 回目から i + m 回目に更新された m + 1 個の閉路について考えるとその中のある組 j, k(j < k) で C_j, C_k は互いに逆向きとなる辺を有している(∃e ∈ C_j, e.rev ∈ C_k) という事実が成り立つことが分かる. この j, k について |L_j| ≧ n * |L_k| / (n - 2) が成り立つことを示す(つまり閉路の最小平均長の絶対値は (n - 2) / n の factor で減少することを示す).
先程同様に C_j と C_k の和集合から打ち消し合う辺を除去したグラフ H について辺コストの総和を考える. このとき j として "C_j, C_k は互いに逆向きとなる辺を有している" という条件を満たすもののうち最大のものを取ってきておくと議論が早い. なぜなら C_k 内にあって C_j 内の辺によって打ち消されないような辺はすべて j 回目の更新時点で残余ネットワーク上に存在することが分かるからである. 言い換えると C_k 内の打ち消されない辺の平均コストは C_j が j 回目の更新時点での最小平均長閉路であるという事実から L_j 以上であると言える.
さらに C_j 内の辺と C_k 内の辺の内少なくとも 1 つは打ち消し合うという事実から |E(H)| ≧ |E(C_j)| + |E(C_k)| - 2 ≧ |E(C_j)| + (n - 2) * |E(C_k)| / n のように辺数をより厳密に評価することができる. 以上を合わせて, L_j * (|E(C_j)| + (n - 2) * |E(C_k)| / n) ≦ L_j * |E(H)| ≦ (グラフ H の辺コストの総和) = L_j * |E(C_j)| + L_k * |E(C_k)| から (n - 2) * L_j / n ≦ L_k, つまり |L_j| ≧ n * |L_k| / (n - 2) が分かる.
ゆえに |L_i| ≧ |L_j| ≧ n * |L_k| / (n - 2) ≧ n * |L_{i+m}| / (n - 2) が成り立つので m 回の反復ごとに少なくとも |L| は (n - 2) / n 倍以下になる. この反復を n 回繰り返すと ((n - 2) / n)^n < e^{1 / 2} < 1 / 2 という事実から, mn 回更新した後には |L| は少なくとも 1 / 2 倍以下になっている.

時間計算量: O(min(m^2 n^2 log (n |c_min|), m^3 n^2 log n)) (ただし c_min は辺コストの最小値を表し, また前者は辺のコストが整数の場合にのみ成り立つ)

コード

template<class edge> class Dinic {
private:
    using Cat = decltype(edge::cap);
    using Cot = decltype(edge::cost);
    vector<vector<edge> >& G;
    const int V;
    vector<int> level, iter;
    void bfs(const int s){
        fill(level.begin(), level.end(), -1);
        queue<int> que;
        level[s] = 0;
        que.push(s);
        while(!que.empty()){
            const int v = que.front();
            que.pop();
            for(const auto& e : G[v]){
                if(e.cap > 0 && level[e.to] < 0){
                    level[e.to] = level[v] + 1;
                    que.push(e.to);
                }
            }
        }
    }
    Cat dfs(const int v, const int t, const Cat f, Cot& res){
        if(v == t) return f;
        for(int& i = iter[v]; i < (int)G[v].size(); ++i){
            edge& e = G[v][i];
            if(e.cap > 0 && level[v] < level[e.to]){
                Cat d = dfs(e.to, t, min(f, e.cap), res);
                if(d > 0){
                    e.cap -= d;
                    G[e.to][e.rev].cap += d;
                    res += d * e.cost;
                    return d;
                }
            }
        }
        return 0;
    }

public:
    Dinic(vector<vector<edge> >& _G) : G(_G), V((int)G.size()), level(V), iter(V){}
    Cot solve(const int s, const int t, Cat f){
        Cot res = 0;
        for(;;){
            bfs(s);
            if(level[t] < 0) return numeric_limits<Cot>::min();
            fill(iter.begin(), iter.end(), 0);
            while(true){
                Cat df = dfs(s, t, f, res);
                if(df > 0){
                    f -= df;
                    if(f == 0) return res;
                }else{
                    break;
                }
            }
        }
    }
};

template<typename CapType, typename CostType> class MinCostFlow
{
public:
    struct edge;
private:
    using Cat = CapType;
    using Cot = CostType;
    const int V;
    const Cot inf;
    vector<vector<Cot> > dp;
    vector<vector<edge*> > prev;
    pair<int, int> minimum_mean_cycle(){
        for(int k = 0; k < V; ++k){
            for(int i = 0; i < V; ++i){
                dp[k+1][i] = inf;
            }
            for(int i = 0; i < V; ++i){
                for(edge& e : G[i]){
                    if(e.cap > 0 && dp[k+1][e.to] > dp[k][i] + e.cost){
                        dp[k+1][e.to] = dp[k][i] + e.cost;
                        prev[k+1][e.to] = &e;
                    }
                }
            }
        }
        if(*min_element(dp[V].begin(), dp[V].end()) == inf) return make_pair(-1, -1);
        int start = -1;
        Cot tnum = inf / V;
        int tden = 1;
        for(int i = 0; i < V; ++i){
            if(dp[V][i] == inf) continue;
            Cot num = -inf / V;
            int den = 1;
            for(int k = 0; k < V; ++k){
                if(dp[k][i] == inf) continue;
                if(num * (V-k) <= den * (dp[V][i] - dp[k][i])){
                    num = dp[V][i] - dp[k][i], den = V - k;
                }
            }
            if(num > -inf / V){
                if((num * tden < den * tnum) || (num * tden == den * tnum && den < tden)){
                    tnum = num, tden = den;
                    start = i;
                }
            }
        }
        return (tnum < 0) ? make_pair(start, tden) : make_pair(-1, -1);
    }
    bool update(Cot& res){
        int level = V, start, length;
        pair<int, int> info = minimum_mean_cycle();
        start = info.first, length = info.second;
        if(start < 0) return false;
        Cat df = numeric_limits<Cat>::max();
        for(int i = 0; i < length; ++i){
            edge *e = prev[level--][start];
            df = min(df, e->cap);
            start = G[e->to][e->rev].to;
        }
        level = V;
        for(int i = 0; i < length; ++i){
            edge *e = prev[level--][start];
            edge *reve = &G[e->to][e->rev];
            e->cap -= df, reve->cap += df, res += df * e->cost;
            start = reve->to;
        }
        return true;
    }
public:
    struct edge {
        int to; Cat cap; Cot cost; int rev;
    };
    vector<vector<edge> > G;
    MinCostFlow(const int node_size)
     : V(node_size), inf(numeric_limits<Cot>::max()),
        dp(V+1, vector<Cat>(V, 0)), prev(V+1, vector<edge*>(V)), G(V){}
    void add_edge(const int from, const int to, const Cat cap, const Cot cost){
        G[from].push_back((edge){to, cap, cost, (int)G[to].size()});
        G[to].push_back((edge){from, 0, -cost, (int)G[from].size() - 1});
    }
    Cot solve(const int s, const int t, Cat f){
        Dinic<edge> dn(G);
        Cot res = dn.solve(s, t, f);
        if(res == numeric_limits<Cot>::min()) return res;
        while(update(res));
        return res;
    }
};
