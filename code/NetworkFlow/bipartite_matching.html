Bipartite Matching

コードについての説明

二部グラフの最大マッチングを求めるアルゴリズム. 例えば n 人の人と m 個の仕事があり, それぞれの人について行うことのできる仕事が分かっていて 1 人が高々 1 つの仕事を行うというような状況を考えます. このとき最大いくつの仕事を行うことができるか？ というような問題をこのアルゴリズムを用いて高速に解くことができます.
人と仕事に対応する n + m 個の頂点と仕事ができるできないに対応する 人 → 仕事 の有向辺からなる2部グラフを考える. ここで超頂点 s,t を source と sink として s → 人, 仕事 → t の辺を先ほどの2部グラフに加えた有向グラフ上で s→t の最大流問題を解く. このときの答えが問題の答えと一致している.
最大流の計算に Dinic 法 を用いると時間計算量は普通に考えると O(n^2m) であるがこの問題に対しては O(msqrt(n)) と高速に動作することが分かっている.
Dinic で一度幅優先探索をしてからもう一度幅優先探索をするまでをフェーズと呼ぶことにする. 辺の容量がすべて 1 であるため s -> t 増加道を見つけたらそのパス上の辺が全て削除される(正確には今後見なくてよい)ので 1 回のステップ(DFS) は O(m) で終了する.
次にステップの総回数について説明する. sqrt(n) 回のステップを終えた時点での増加道の長さは sqrt(n) 以上であり, かつ 「点素な増加道の数頂点数 <= 頂点数/増加道の長さ」 が成り立つので残りの(マッチングとの対称差の意味での)点素な増加道の数は sqrt(n) で抑えられる. よって残りのステップ数は O(sqrt(n)) であることが言え, 合計で O(sqrt(n)) 回となる. よって計算量は上述の通り O(m sqrt(n)) となる.
2 部グラフに 2 つの超頂点を足した形のグラフがすべて O(m sqrt(n)) の計算量で解けるわけではなく, 頑張って 2 部グラフ型だが計算量が O(m sqrt(n)) とならない例を構成してみた(コード).
(補足)
実質やっていることは同じことだが, 二部グラフのマッチングは増加道アルゴリズムを用いても解くことができる.
最大流最小カット定理のように二部グラフの最大マッチング問題を整数計画問題に書き下し、その緩和問題である線形計画問題の双対を取って解に整数制約を加えると最小頂点被覆問題になる. ここで現れる行列が完全単摸性を持つ(無向2部グラフの接続行列なので)ことが確認できるので強双対性と合わせて最適値は等しくかつ解ベクトルは整数となる.
つまり最大マッチングのサイズは最小頂点被覆のサイズに等しいことが言える. これは Kőnig の定理と呼ばれ、 これは Hall の結婚定理と等価の定理である.
二部グラフの左側, 右側を A, B とし、マッチングの済んだ(A-B間の)残余グラフ上で A のマッチングされていない頂点からたどれる頂点の集合を L とする. このとき (A-L)∪(B∩L) が最小頂点被覆に対応する. もし被覆されてない e=(a,b)(a ∈ A∩L, b ∈ B-L) があったとする. (a, b) がマッチングに含まれないとすると a から b がたどれてしまい b ∈ L でダメ, 含まれるとすると a ∈ L から b から a にたどったことになり b ∈ L が成り立ちこれもダメ.

(コンストラクタ)
BM(u, v): u が人の数, v が仕事の数
(関数)
add_edge(u,v): 人 u と仕事 v の間に辺を追加する
solve(): 最大マッチングを求める
allocate(): solve() の後に行うと人 i に割り当てられた仕事のインデックスを返す(なければ-1)

時間計算量: O(msqrt(n))

コード

class BM {
private:
    struct edge {
        int to,cap,rev;
    };
    int U,V;
    vector<vector<edge> > G;
    vector<int> level;
    vector<int> iter;

public:
    BM(int u,int v) : U(u), V(v), G(U+V+2){
        for(int i = 0; i < U; i++){
            init_edge(0,i+1);
        }
        for(int i = 0; i < V; i++){
            init_edge(U+i+1,U+V+1);
        }
    }
    void init_edge(int from,int to) {
        G[from].push_back((edge){to,1,(int)G[to].size()});
    	G[to].push_back((edge){from,0,(int)G[from].size()-1});
    }
    void add_edge(int from,int to) {
        from += 1,to += U+1;
    	G[from].push_back((edge){to,1,(int)G[to].size()});
    	G[to].push_back((edge){from,0,(int)G[from].size()-1});
    }
    void bfs(int s) {
        fill(level.begin(),level.end(),-1);
    	queue<int> que;
    	level[s] = 0;
    	que.push(s);
    	while(!que.empty()){
            int v = que.front();
            que.pop();
            for(auto& e : G[v]){
                if(e.cap > 0 && level[e.to] < 0){
                    level[e.to] = level[v] + 1;
                    que.push(e.to);
                }
            }
        }
    }
    int dfs(int v,int t,int f) {
        if(v==t){
            return f;
        }
        for(int &i = iter[v];i<(int)G[v].size();i++){
            edge &e = G[v][i];
            if(e.cap > 0 && level[v] < level[e.to]){
                int d = dfs(e.to,t,min(f,e.cap));
                if(d>0){
                    e.cap -= d;
                    G[e.to][e.rev].cap += d;
                    return d;
                }
            }
        }
        return 0;
    }
    int solve() {
        level.resize(U+V+2), iter.resize(U+V+2);
        int flow = 0;
        int s=0,t=U+V+1;
        for(;;){
            bfs(s);
            if(level[t] < 0) return flow;
            fill(iter.begin(),iter.end(),0);
            int f;
            while((f=dfs(s,t,numeric_limits<int>::max())) > 0){
                flow += f;
            }
        }
    }
    vector<int> allocate() {
        vector<int> res(U, -1);
        for(int i = 0; i < U; i++){
            for(auto& e : G[i+1]){
                if(e.cap == 0 && e.to != 0){
                    res[i] = e.to-U-1;
                    break;
                }
            }
        }
        return res;
    }
};
