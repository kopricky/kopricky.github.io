quadratic decomposition

コードについての説明

segment tree ではうまく処理できないような区間に対するクエリ、例えば区間 [l,r) に値 val がいくつ含まれているか や 区間 [l,r) 内の k 番目に小さい数はいくつか というような問題に対して有効なデータ構造.
全体で要素数 n 個の配列を要素数 B 個のブロックに分ける. となり、区間 [l,r) について完全に区間に含まれるブロックが O(n/B) 個, 区間の両端で中途半端にブロックに含まれる要素が O(B) 個となり, 各ブロックおよび要素にを O(1) で処理できると考えた場合一つのクエリを処理するのにかかる計算量は O(n/B+B) である. またここで B = sqrt(n) とすると O(sqrt(n)) となる.
各ブロックについてソートを行うようなクエリ((例)区間 [l,r) 内の k 番目に小さい数はいくつか)の場合はブロックを sqrt(nlog(n)) ぐらいで取ったほうが良い(ブロックの計算量とはみ出た要素の計算量とのトレードオフ)
TL が厳しい場合は, ブロックサイズをいろいろ変更して提出したほうがよさそう.

時間計算量: 構築 O(n^2/B), クエリ O(n/B+B)

コード

template<typename T> class QuadraticDecomposition{
public:
    vector<T> data;
    int sz,backet_size,backet_cnt;
    vector<T> backet;
    QuadraticDecomposition(vector<T>& v) : data(v), sz((int)data.size()),
        backet_size((int)sqrt(sz)), backet_cnt(sz / backet_size + 1), backet(backet_cnt, 0){
        for(int i = 0; i < sz; i++){
            backet[i / backet_size] += data[i];  //例えば和を持たせる処理
        }
    }
    // [l.r)
    T query(int l, int r){
        T res = 0;
        int backet_index, data_index;
        // 同じバケット内にある
        if(l / backet_size == (r-1) / backet_size){
            for(data_index = l; data_index < r; data_index++){
                res += data[data_index];
            }
        }else{
            // 完全に被っているバケット
            for(backet_index = (l + backet_size - 1) / backet_size; backet_index < r / backet_size; backet_index++){
                res += backet[backet_index];
            }
            // 中途半端な左側のバケット
            if(l % backet_size){
                for(data_index = l; data_index < backet_size*(l / backet_size + 1); data_index++){
                    res += data[data_index];
                }
            }
            // 中途半端な右側のバケット
            if(r % backet_size){
                for(data_index = r - r % backet_size; data_index < r; data_index++){
                    res += data[data_index];
                }
            }
        }
        return res;
    }
};
