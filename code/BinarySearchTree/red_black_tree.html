Red Black Tree

コードについての説明

Red Black Tree(赤黒木) は値の検索, 挿入, 削除などを最悪時 O(log n) (要素数を n とする) 回の要素比較で達成する平衡二分探索木であり, C++ の std::set, std::map などにも用いられている代表的なデータ構造である.
赤黒木は常に次の 3 つの条件を保つ. (1) 各ノードには赤もしくは黒の色がつけられている, (2) 赤色のノードの子は必ず黒色のノードである, (3) あるノードからそのノードの子孫である葉までのパス上に含まれる黒色のノードの数は葉の選択に依らず一定である.
この 3 つの条件からあるノードからそのノードの子孫である葉までの距離について (最大距離) が (最小距離) の 2 倍で抑えられることが言える. 文脈によって "根は黒", "葉は黒", "葉以外のノードはちょうど 2 つの子を持つ" などの条件が入ることもあるが本質的ではない.
赤黒木は insert/delete ベースと merge/split ベースの 2 つがあり std::set, std::map などは前者であるが, 下の実装では merge/split を行いたいので merge/split ベースにしている.
