Chinese Remainder Theorem

コードについての説明

Chinese Remainder Theorem(中国剰余定理) は p_1 と p_2 が互いに素で x = q_1 (mod p_1), x = q_2 (mod p_2) が成り立つとき, これを満たす x は [0, p_1*p_2) にただ一つ存在するという定理のことであり, 以下の実装は実際にその唯一の解を求めるアルゴリズムである
アルゴリズムは結局式変形すると拡張ユークリッドの互除法(extgcd)を解くことと同じであることが分かる. 以下の実装では一般の場合(法が互いに素でない場合)に拡張したものも含まれています.

(関数)
CRT(a1, a2): a1(q_1, p_1), a2(q_2, p_2) を引数とし, x = q (mod p) を満たす (q, p) を返す

時間計算量: O(log p) (p は法の値)

コード

template<typename T>
T gcd(T a,T b)
{
    if(a % b == 0){
        return b;
    }else{
        return gcd(b,a%b);
    }
}

template <typename T>
void extgcd(T a, T b, T& x, T& y)
{
    if(b != 0){
        extgcd(b,a%b,y,x);
        y -= (a/b)*x;
    }else{
        x = 1;
        y = 0;
    }
}

// mod が long long のときに以下のようなものが使えるらしい (a * b % mod)
long long mod_prod(long long a, long long b, long long mod) {
    long long res = (a * b - (long long)((long double)a / mod * b) * mod) % mod;
    return res < 0 ? res + mod : res;
}

//modが10^6より大きい値を扱うときは m1 * x を__int128にキャストする(か上の mod_prod を用いる)(10^12とかまで可能)
//(value, mod)
template <typename T>
pair<T, T> CRT(const pair<T, T>& a1, const pair<T, T>& a2)
{
    const T v1 = a1.first, m1 = a1.second;
    const T v2 = a2.first, m2 = a2.second;
    if(v1 == v2) return make_pair(v1,m1*m2);
    T x, y;
    extgcd(m1, m2, x, y);
    x *= (v2 - v1), y *= (v2 - v1);
    const T m = m1 * m2;
    return make_pair((((m1 * x + v1) % m) + m) % m, m);
}
